## Epic Summary

Epic 4 delivered the full portfolio and project showcase experience:

- Portfolio overlay redesigned from BulletPoints grid to a semantic ProjectCard-driven single-column layout with image-first hierarchy, TechBadge rows, and ExternalLinkButton pairs for Live Demo and GitHub links.
- Portfolio data refreshed with current copy, tech stacks, and the Money Manager URL cutover to mm.wsapz.com. Story 4.2 achieved this with zero component code changes — purely data file edits.
- Proof-to-skills reverse navigation implemented, extending the Story 3.2 close → grid → open choreography to work in both directions (About → Portfolio and Portfolio → About).
- My Approach card and ApproachContent overlay added, expanding the card grid from five to six cards with a new interactive entry point.
- Bidirectional navigation verified end-to-end with headless Chrome CDP interaction checks across desktop, tablet, and mobile.

All 3 stories are marked `done` in `sprint-status.yaml`. Across Stories 4.1–4.3, 15 review findings were recorded; all actionable findings were resolved before close. Epic 4 added 6 new deferred items, all recorded as non-blocking.

## What Went Well

- **Scope discipline was exceptional across all three stories.** Story 4.1 built the UI and minimum data contract, 4.2 refreshed data only, 4.3 added reverse navigation and Approach. Zero scope leakage between stories.
- **Cross-epic contract preservation worked flawlessly.** Stable `skillId` and `projectId` values from Epic 3 survived all three stories without incident.
- **Data-driven architecture proved its ROI.** Story 4.2 refreshed all portfolio content by editing only data files — zero component changes. The data-first investment from Story 4.1 paid off immediately.
- **Zero blockers, zero reopens, zero rollbacks.** Every story went from `in-progress` to `done` without emergency fixes.
- **Six-card grid expansion landed cleanly.** Story 4.3 correctly identified the saturated 3x2 layout, added the Approach card without breaking Contact, and updated responsive layouts.
- **Bidirectional navigation completed and verified.** Both About → Portfolio (Epic 3) and Portfolio → About (Epic 4) now use the same choreography pattern.
- **Review quality remained high.** Reviews caught real type-safety issues (lazy import assertions, defensive fallbacks on mapping lookups) and data-integrity concerns.

## Challenges

- **Card layout needs rework.** The six-card layout breaks card symmetry — the Contact card is disproportionately large. The icon is visually orphaned from the card title. The hero image card and "Warrick Smith Developer" card should be consolidated into one adaptive card, reducing back to five main cards. Warrick wants the four interactive cards to be equal size, with the merged hero/identity card independently adaptive.
- **Epic 3 retro follow-through was weak.** Only 1 of 8 action items was fully completed. Three critical items (project-context.md update, planning-doc language fix, live verification debt) remain untouched through two consecutive epics.
- **Documentation drift is now chronic.** `project-context.md` still references styled-components as the primary styling system. All three Epic 4 stories explicitly warned about this, creating recurring story-prep waste.
- **Deferred work pile keeps growing but never shrinks.** Epic 4 added 6 new deferred items. The deferred-work.md file is maintained, but items go in and almost never come out. Warrick identified this as a breakdown in the retrospective process — deferred items must be allocated to specific stories or a new epic must be created to address them.
- **Missing core skill badges remain unresolved.** React, TypeScript, Tailwind CSS, Next.js, Fastify, and Appwrite are not in the skills data. Now that bidirectional navigation is live, the gap is visible to visitors.
- **Card ID magic numbers throughout codebase.** `renderChildDiv.tsx` and `MainPage.tsx` mix raw numbers (`3`/`4`/`5`/`6`) with partial named constants.
- **Arbitrary responsive breakpoints creeping in.** ApproachContent uses `min-[860px]` which doesn't align with project design tokens.

## Key Decisions Made

- **Card consolidation for Epic 6.** Hero image (Card 1) and identity text (Card 2) will merge into one adaptive card. Large screen: image left, text right. Mid screen: image above, text below. Mobile: text replaces or sits beside small image. This reduces the main grid back to five cards.
- **Card sizing constraint.** The merged hero/identity card is independently adaptive in size. The remaining four interactive cards (About, Portfolio, Approach, Contact) must be equal size.
- **Card preview layout rework.** Icon must be inline with card title text, not visually orphaned above it. Card text should be vertically centered or top-aligned.
- **Deferred items must get dispositions.** Every deferred item from Epics 1–4 must be assigned to a specific story, collected into a cleanup story, or explicitly accepted as permanent debt with Warrick's sign-off. No item leaves this retro without a disposition.
- **Retro action items must become owned tasks or stories within one sprint.** If an action item isn't turned into a concrete task, it gets escalated.
- **All missing core skill badges will be added and linked to projects.** New entries: React, TypeScript, Tailwind CSS, Next.js, Fastify, Appwrite. Updates: PostgreSQL → race-day, MongoDB → music-manager.
- **No production deploy until all epics are complete.** The entire project ships as a single production release after Epic 6.
